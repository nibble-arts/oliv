Plugin-System
-------------

Es gibt Bereiche, deren Funkionalität über Plug-ins erweitert werden kann.

Renderer:
---------
Die Renderengine verbindet das Template mit dem Content und erstellt gültigen Html-Code, der zurückgegeben wird. Es können in beiden Bereichen Definitionen für die Ausgabesteuerung existieren. Bei gleichen Definitionen überschreiben die Content-Inhalte die zuvor definierten Template Einträge.
Der Renderer vereinigt die Parameter-Einträge und ruft die definierten Module auf. Im Rendervorgang werden die Tags analysiert. Dabei greift er auf eine Liste von definierten Tags zurück. Während des Preprocessing werden die Plugins in der Liste registriert. Dabei können vordefinierten Einträge überschrieben und mit neuen Funktionen für den jeweiligen Modulaufruf mit eigenen Funktionen belegt werden.

Plugins werden im Unterverzeichnis plugins/ abgelegt und vom Preprozessor geladen. Dabei werden die Metadaten eingelesen und registriert. Es können unterschiedliche Pluginzweige verwendet werden. So gibt es einen Zweig für Rendering Erweiterungen. Die Standard Tags des Textrenderngs sind ebenfalls als Plugin angelegt.

Render Plugin Definition
------------------------

<plugin>
	<name>pluginName</name>
	<class>render</class>
	<tags>
		<tagName>functionName</tagName>
			.....
	</tags>
</plugin>

Den verschiedenen Tags können Funktionsaufrufe aus der Plugin-Klasse zugeordnet werden.

Der zweite große Zweig ist die Suche. Hier können Module eigene Suchfunktionen registrieren, wie zum Beispiel die Datenbank.

Search Plugin Definition
------------------------

<plugin>
	<name>pluginName</name>
	<class>search</class>
	<search>functionName</tagName></search>
</plugin>
